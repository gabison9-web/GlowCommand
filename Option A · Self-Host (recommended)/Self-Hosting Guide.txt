  STREAMELEMENTS · OPTION A — Self-Host (recommended)
  Upload widget files to a free static host, use the public URL
  in StreamElements as a Custom URL widget.

WHY SELF-HOST
───────────────────────────────────────────────────────────────

  • All features work — including live Twitch chat pulse
  • The widget behaves identically across OBS / Streamlabs / SE
  • You can update the widget by re-uploading new versions
  • Free hosting takes 5 minutes to set up


FILES IN THIS FOLDER
───────────────────────────────────────────────────────────────

  📄 widget.html
  📁 assets/  styles.css · widget.js · icons.js
  📁 config/  commands.example.json

These are the same files used by OBS — just hosted on the web
so StreamElements can load them.


THREE FREE HOSTING OPTIONS — pick whichever feels easiest
───────────────────────────────────────────────────────────────

🟢 OPTION 1: Netlify Drop  (easiest — drag-and-drop, no account)

   1. Open this URL in your browser:    https://app.netlify.com/drop
   2. Drag this entire folder onto the page
   3. After upload, Netlify gives you a public URL like:
        https://random-name-12345.netlify.app/
   4. Your widget URL is:
        https://random-name-12345.netlify.app/widget.html
   5. Skip to "USE THE URL IN STREAMELEMENTS" below.


🟢 OPTION 2: GitHub Pages  (free, lasts forever, requires GitHub account)

   1. Create a free GitHub account at github.com if you don't have one.
   2. Create a new public repository (e.g. "my-stream-widget")
   3. Upload these files (drag-and-drop on github.com works):
        • widget.html
        • assets/  (whole folder)
        • config/
   4. In your repo: Settings → Pages
   5. Under "Source", pick "Deploy from a branch", branch = main, folder = /
   6. Save. After 1 minute your URL will be:
        https://YOUR_USERNAME.github.io/my-stream-widget/widget.html


🟢 OPTION 3: Cloudflare Pages, Vercel, or your own webhost

   Any static-file host works. Upload all the files preserving
   the folder structure (keep assets/ and config/ as folders).


USE THE URL IN STREAMELEMENTS
───────────────────────────────────────────────────────────────

Once you have your hosted URL (e.g. https://...netlify.app/widget.html):

   1. Go to:  streamelements.com → My Overlays
   2. Click "New Overlay" (or open an existing one)
   3. Click + → "Static / Custom" → "Browser Source" or "Custom URL"
   4. Paste your URL with parameters appended:

        https://yourhost.com/widget.html?theme=pastel-dream&mode=rotating&rotation=4

   5. Set the source dimensions:

        Rotating mode:    720 × 360
        Grid mode:        720 × 540
        Scroll mode:      360 × 480
        Compact mode:     560 × 80

   6. Position the widget on your overlay canvas.
   7. Save.


URL PARAMETERS REFERENCE
───────────────────────────────────────────────────────────────

Combine these with & in the URL:

  &theme=NAME           pastel-dream | night-glow | cozy-warm | nature-calm
  &mode=NAME            rotating | grid | scroll | compact
  &rotation=N           seconds per card (default 4.5)
  &label=TEXT           panel header text (use + for spaces)
  &pinned=NAME          pin a command at the top
  &chat=1&channel=YOU   live Twitch chat pulse (anonymous)
  &schedule=1&every=180&show=15   pop-in schedule
  &cotd=NAME            "today's pick" featured command
  &peek=1               show coming-up chip
  &catrot=1             cycle by category with banner
  &sound=1              soft chime on rotation

Full list: 4 - Customize & Advanced / Advanced Features /
           All URL Parameters.txt


UPDATING / EDITING YOUR COMMANDS
───────────────────────────────────────────────────────────────

To change your !commands list:

   1. Open assets/widget.js in any text editor
   2. Edit the DEFAULTS.commands array (see "How to Edit
      Commands.txt" in the Customize folder)
   3. Re-upload to your host (Netlify / GitHub / etc.)
   4. In StreamElements, click "Refresh" on the widget so it
      pulls the new version.


TROUBLESHOOTING
───────────────────────────────────────────────────────────────

Widget loads but is blank:
  • Make sure ALL the asset files (assets/styles.css, etc.) are
    accessible at the same hosted location. Test by opening:
      https://yourhost.com/assets/styles.css   ← should load
  • Check browser console (F12) on your hosted URL for 404s.

Widget loads at home but not in StreamElements overlay:
  • SE may be blocking the URL — try a different hosting option.
  • Check the URL is exactly correct (case-sensitive on most hosts).

Live chat pulse not working:
  • Self-hosted should work fully. Check your channel parameter
    matches your Twitch username (lowercase, no @).
